Forge Plugin — Story Map

Процессы → шаги → статус. Сущности → атрибуты. Дыры.

Процессы

Session Start

Открытие Claude Code
→
✓ Hook загружает using-forge скилл
✓ Инжект L0 из .forge/index.yml (~200 токенов)
✓ Keyword matching → подсказка скилла
✗ Автозагрузка L1 по контексту задачи

Brainstorm

Новая фича / изменение
→
✓ 3 агента: аналитик, технолог, критик
✓ Требования: must-have vs nice-to-have
✓ Дизайн-документ с acceptance criteria
✓ GATE: дизайн одобрен пользователем

Plan

После одобрения дизайна
→
✓ Декомпозиция на задачи по 2-5 мин
✓ Каждая задача: файлы + тест + код + проверка
✓ План в .forge/plans/*.md

Execute

Выполнение плана
→
✓ Свежий субагент на каждую задачу
✓ TDD: RED → GREEN → REFACTOR
✓ Spec review + Quality review
✓ Documenter обновляет .forge/
✗ Auto skill chaining (сейчас вручную)

Debug

Баг / ошибка
→
✓ Phase 1: Расследование (обязательно!)
✓ Phase 2: Гипотеза
✓ Phase 3: Верификация
✓ Phase 4: Фикс
✓ Запись Dead End при провале

Sync & Finish

После работы
✓ Git diff → обновление docs
✓ Запись decisions, dead-ends, learnings
✓ Code review → merge/PR
✗ Автоизвлечение learnings из коммитов
✗ Автоизвлечение dead-ends из логов
Сущности

Skill

Воркфлоу-паттерн для Claude. 23 штуки — от TDD до деплоя.
implemented
Атрибуты
  • namekebab-case: test-driven-development, systematic-debugging...
  • triggerbug/fix → debugging, test → TDD, plan → writing-plans
  • typerigid (шаг влево нельзя) | flexible (адаптируй)
  • stagepre-implementation | execution | verification
Связи
  • вызывается из Session
  • управляется Review Gate
  • содержится в Skill Catalog

Session

Рабочая сессия Claude с пользователем. От открытия до закрытия.
implemented
Атрибуты
  • goalЦель — что хотим сделать сегодня
  • doneВыполненные шаги
  • nowТекущая задача
  • nextСледующий шаг
  • errorsОшибки за сессию
Связи
  • загружает Context Layer
  • активирует Skill
  • записывает в Journal

Context Layer

Три уровня контекста. L0 всегда, L1 по тегам, L2 редко. Экономит токены.
implemented
Атрибуты
  • L0~200 токенов, ВСЕГДА. Цель, стадия, каталог
  • L1~500-2000 по тегам. map, status, decisions, dead-ends
  • L2Без лимита, РЕДКО. Полные spec.yml, планы
  • budgetТипичная задача: 400-700 токенов
Связи
  • инжектируется Hook'ом в каждый промпт
  • хранит Dead Ends, Decisions, Learnings

Plan

Детальный план фичи. Создаётся ПОСЛЕ brainstorm. Задачи по 2-5 минут.
implemented
Атрибуты
  • goalЧто реализуем — одно предложение
  • architectureКомпоненты, потоки данных
  • tasksСписок: файлы + тест + код + проверка
  • file.forge/plans/YYYY-MM-DD-feature.md
Связи
  • создаётся после Brainstorm (gate!)
  • содержит Tasks
  • выполняется Subagent'ами

Task

Атомарная единица работы. 2-5 минут. Один таск = один коммит.
implemented
Атрибуты
  • filesКакие файлы создать/изменить
  • failing_testЧто тестировать ПЕРВЫМ (RED)
  • minimal_codeМинимум чтобы тест прошёл (GREEN)
  • verificationКак убедиться что работает
Связи
  • принадлежит Plan
  • выполняется Subagent (implementer)
  • проходит Review Gate

Subagent

Свежая сессия Claude на одну задачу. Чистый контекст, без загрязнения.
implemented
Атрибуты
  • roleimplementer | spec-reviewer | quality-reviewer | documenter
  • briefПолный контекст — субагент не видит историю
  • stack_hintspython.md, react.md, go.md, typescript.md
  • modelimplementer: inherit, documenter: sonnet
Связи
  • выполняет Task
  • проверяется Review Gate
  • оркестрируется Controller

Review Gate

Обязательная проверка. Пропустить НЕЛЬЗЯ. Блокирует прогресс.
implemented
Атрибуты
  • typedesign-approved | test-passing | spec-compliance | code-quality
  • blockingВСЕГДА true — обхода нет
  • enforcementRed flags таблица ловит попытки обойти
Связи
  • блокирует Task, Branch merge, Plan execution
  • выполняется Subagent (reviewer) или авто (тесты)

Дыры и недоделки
!

Marketplace API

Discover ищет только локально. Удалённый поиск и автоустановка не работают.

!

Skill Chaining

Пользователь вручную вызывает следующий скилл. Нет автоматической цепочки brainstorm → plan → execute.

!

Learning Extraction

Не извлекает инсайты автоматически из ошибок, коммитов, паттернов.

!

Cross-Project Patterns

Нет переноса learnings и dead-ends между проектами.

!

Skill Auto-Detection

Только keyword matching. Нет семантического понимания когда какой скилл нужен.
