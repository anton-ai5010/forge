# Обзор репозиториев — Оценка и вердикты

---

## 🟢 СТАВИТЬ

### 1. Claude Mem (39.6K ⭐) — `thedotmack/claude-mem`
**Что делает**: Плагин который автоматически захватывает всё что Claude делает в сессии, сжимает AI-ом, и инъектирует релевантный контекст в будущие сессии. SQLite + полнотекстовый поиск. Worker-сервис крутится в фоне.

**Оценка**: Killer feature. Прямо решает проблему потери контекста. 39.6K звёзд не случайно. Активно развивается — v10.6, 1500+ PR-ов. AGPL лицензия, требует Node 18+ и PM2, на Windows глючит. На Ubuntu работает нормально.

**Роль в ADS**: Заменяет самописный project-context скилл и session-start хук. Делает то же самое, только автоматически и умнее. Становится Memory Layer в архитектуре.

**Установка**:
```bash
/plugin marketplace add thedotmack/claude-mem
/plugin install claude-mem
```

---

### 2. Everything Claude Code (17K+ ⭐) — `affaan-m/everything-claude-code`
**Что делает**: 38 специализированных агентов, 156 скиллов, 72 команды, автоматические хуки. Полная система для разработки.

**Оценка**: Один из лучших "всё-в-одном" пакетов. Opinionated — навязывает определённый workflow. Хорошо написан. Может конфликтовать с другими плагинами если ставить вместе.

**Роль в ADS**: Альтернатива кастомному плагину. Если не хочешь собирать своё — поставь его целиком.

---

### 3. Superpowers — `obra/superpowers`
**Что делает**: Набор core competencies для software engineering — планирование, ревью, тестирование, дебаг. Включает systematic-debugging, root-cause-tracing, test-driven-development, subagent-driven-development.

**Оценка**: Качественные скиллы от опытного разработчика. Не раздутый, сфокусированный набор.

**Роль в ADS**: systematic-debugging и subagent-driven-development встроить в Inspector. Хорошо дополняет архитектуру.

---

### 4. Awesome Claude Code — `hesreallyhim/awesome-claude-code`
**Что делает**: Курированный список лучших инструментов с авторскими обзорами.

**Оценка**: Не плагин, а навигатор — лучший способ найти новые инструменты. Автор реально тестирует и даёт честные оценки.

**Действие**: Закладка, не установка. Возвращаться регулярно за обновлениями.

## 🟡 ОТЛОЖИТЬ

### 5. n8n-MCP — `czlonkowski/n8n-mcp`
**Что делает**: MCP-сервер для n8n — Claude Code может создавать, редактировать и управлять n8n workflows прямо из терминала.

**Оценка**: n8n — мощный инструмент для автоматизации бизнес-процессов. Если подключить к CAME BOT — Claude сможет строить автоматизации: "заказ пришёл → проверь остатки → уведоми в Telegram → закупи если мало". Но нужна дополнительная инфраструктура — запущенный n8n.

**Когда ставить**: Когда дойдёшь до автоматизации бизнес-процессов CAME BOT.

---

### 6. LightRAG (31.5K ⭐) — `hkuds/lightrag`
**Что делает**: RAG-фреймворк который строит knowledge graph из документов и использует мульти-модальный поиск (local, global, hybrid) для запросов. Извлекает сущности и связи из текста. Web UI для визуализации графа.

**Оценка**: Не плагин для Claude Code, а самостоятельный фреймворк. Мощный, академический (EMNLP 2025), но требует: LLM с 32B+ параметров, embedding модель, инфраструктуру. Для YARICK потенциально интересен — скормить все статьи о футбольных предсказаниях и спрашивать "какие фичи лучше всего работают для OU2.5".

**Когда ставить**: Когда нужна серьёзная работа с базой знаний для YARICK.

---

### 7. GSD — Get Shit Done — `gsd-build/get-shit-done`
**Что делает**: Система автоматизированной разработки — описываешь что нужно, получаешь production-grade результат. Код, контент, дизайн, планирование.

**Оценка**: Концептуально близко к Foreman — оркестрация задач. Но менее кастомизируемый. Ready-to-go вариант.

**Действие**: Изучить их оркестрацию, сравнить с Foreman. Если их подход лучше — позаимствовать паттерны.

---

### 8. UI UX Pro Max — `nextlevelbuilder/ui-ux-pro-max-skill`
**Что делает**: Скилл для UI/UX дизайна — паттерны, best practices, компонентный подход.

**Оценка**: Полезный скилл для фронтенда. Но ClaudeKit web-dev-tools покрывает ту же область с бОльшим охватом. Дублирование.

**Действие**: Попробовать если ClaudeKit не устроит.
