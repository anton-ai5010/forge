Главный процесс: от идеи до мержа

Session Start

Открытие Claude Code
→
✓ Hook загружает using-forge
✓ Инжект L0 (~200 токенов)
✓ Keyword → skill hint
✗ Автозагрузка L1 по контексту

Brainstorm

Новая фича
→
✓ 3 агента: аналитик, технолог, критик
✓ Требования + acceptance criteria
✓ GATE: дизайн одобрен

Plan

После дизайна
→
✓ Декомпозиция на задачи 2-5 мин
✓ Каждая: файлы + тест + код + проверка
✓ План в .forge/plans/*.md

Execute

Субагенты работают
→
✓ Свежий субагент на задачу
✓ TDD: RED → GREEN → REFACTOR
✓ Spec review + Quality review
✗ Auto skill chaining

Debug

Баг / ошибка
→
✓ Phase 1: Расследование
✓ Phase 2-3: Гипотеза + верификация
✓ Phase 4: Фикс
✓ Запись Dead End при провале

Sync & Finish

Завершение
✓ Git diff → обновление docs
✓ Запись decisions, dead-ends
✓ Code review → merge/PR
✗ Автоизвлечение learnings

!

Marketplace API

/forge:discover ищет только по локальным маркетплейсам. Удалённый поиск и автоустановка — нет.

!

Skill Chaining

Нет автоматической цепочки brainstorm → plan → execute. Пользователь вручную вызывает каждый следующий скилл.

!

Learning Extraction

Не извлекает инсайты из ошибок, коммитов и паттернов автоматически. Только ручная запись через /forge:sync.

!

Cross-Project Patterns

Каждый .forge/ изолирован. Нет переноса learnings и dead-ends между проектами.

!

Skill Auto-Detection

Только keyword matching в context-inject.sh. Нет семантического понимания когда какой скилл нужен.

!

Авто-загрузка L1

L1 контекст не загружается автоматически по задаче. Claude должен сам запросить нужный файл.

!

Auto Red Zone Detection

Red zones определяются вручную при /forge:init. Нет автоматического анализа критичности файлов.
