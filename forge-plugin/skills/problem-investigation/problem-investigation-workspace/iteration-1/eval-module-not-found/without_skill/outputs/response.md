Это классическая проблема с путями в Node.js. Давай разберемся.

Покажи мне несколько вещей:

1. Точный текст ошибки целиком (со стектрейсом)
2. Какой файл делает `require('../lib/utils')` или `import ... from '../lib/utils'`?
3. Структуру директорий — запусти:

```bash
ls -la lib/
ls -la lib/utils*
```

Пока ты это собираешь, вот самые частые причины этой ошибки:

**1. Расширение файла.** Node ищет `utils.js` (или `.ts`, `.mjs`), а у тебя файл может называться `utils.ts` без настроенного резолва. Проверь что в `package.json` или конфиге бандлера прописаны нужные расширения.

**2. Регистр имени файла.** На Linux файловая система case-sensitive. Если файл называется `Utils.js`, а импорт `../lib/utils` — модуль не найдется. Проверь:

```bash
ls -la lib/ | grep -i utils
```

**3. Путь относительно скомпилированного файла, а не исходника.** Если используется TypeScript или бандлер, `require('../lib/utils')` резолвится относительно скомпилированного файла в `dist/` или `build/`, а не относительно исходника в `src/`. Проверь, есть ли `lib/utils` рядом с выходным файлом.

**4. tsconfig paths / aliases.** Если используешь path aliases в `tsconfig.json`, нужно чтобы рантайм тоже их понимал (через `tsconfig-paths`, `tsc-alias`, или настройку бандлера).

**5. Симлинки / node_modules.** Иногда `npm link` или монорепо workspace создают ситуацию, где реальный `__dirname` отличается от ожидаемого.

Скинь вывод команд и полный текст ошибки — дальше будет понятнее.
