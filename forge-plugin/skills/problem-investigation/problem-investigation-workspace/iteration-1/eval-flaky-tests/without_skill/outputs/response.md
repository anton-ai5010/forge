Понял, давай разберемся. Раз ты ничего не менял, причина скорее всего внешняя. Давай начнем с диагностики.

Сначала проверю, действительно ли не было изменений в коде:

```
git status
git diff HEAD~1
```

Потом запущу тесты чтобы увидеть конкретные ошибки:

```
npm test
```

Пока я это проверяю — вот основные причины, почему тесты могут сломаться "сами по себе":

1. **Зависимости обновились** — если в package.json есть `^` или `~`, `npm install` мог подтянуть новую минорную/патч версию
2. **Время/дата** — тесты завязаны на текущую дату, таймзону или таймауты
3. **Кэш/state** — остался мусор от прошлого запуска (tmp файлы, базы, порты)
4. **Окружение** — обновился Node.js, ОС, или изменились env-переменные
5. **Flaky тесты** — гонки, зависимость от порядка выполнения, сетевые вызовы

Покажи мне вывод падающих тестов — по ошибкам будет понятнее что именно случилось.
