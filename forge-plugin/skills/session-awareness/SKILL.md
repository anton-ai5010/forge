# Session Awareness

Ты — рабочая память Claude. Контекст сжимается, сообщения теряются,
но docs/index.md живёт. Ты читаешь его перед каждым ответом и обновляешь
после каждого действия. Это делает тебя "помнящим".

## Когда активируется

КАЖДЫЙ запрос если существует `docs/index.md`. Без исключений.

## Два режима

### Быстрый (~400 токенов) — для простых запросов

"Переименуй переменную", "добавь комментарий", "запусти тесты"

1. Прочитай `docs/index.md`
2. Проверь `Session.Now` — не противоречит ли запрос текущей задаче?
3. Выполни запрос
4. Обнови `Session.Done` и `Session.Now` в index.md

### Полный (~800-1200 токенов) — для задач

"Почини баг", "добавь фичу", "разберись почему падает"

1. Прочитай `docs/index.md`
2. Определи тему задачи (auth, database, ui, api и т.д.)
3. Проверь `ls docs/dead-ends/` — есть ли файл по этой теме?
   - Если да — прочитай, НЕ повторяй провальные подходы
4. Если архитектурный выбор — прочитай `docs/decisions.md`
5. Если нужно понимание состояния — прочитай `docs/status.md`
6. Выполни задачу
7. Обнови index.md

### Как определить режим

| Запрос | Режим |
|---|---|
| Одно простое действие, нет риска ошибки | Быстрый |
| Исправление бага | Полный |
| Новая фича или изменение поведения | Полный |
| Рефакторинг | Полный |
| Вопрос про проект | Быстрый (index.md уже содержит ответ) |
| Не уверен | Полный (лучше прочитать лишнее чем пропустить) |

## Что читать

```
ВСЕГДА:
  docs/index.md                    # ~400 токенов

ПО НЕОБХОДИМОСТИ:
  docs/dead-ends/<тема>.md         # ~100-300 токенов, только релевантный файл
  docs/decisions.md                # ~300 токенов, при архитектурных выборах
  docs/status.md                   # ~200 токенов, при вопросах о состоянии
  docs/journal.md                  # ~150 токенов (верхняя запись), для контекста
```

НЕ ЧИТАЙ всё подряд. index.md содержит ссылки — иди по ним только когда нужно.

## Что писать

### index.md — обновляется ЖИВЬЁМ

| Момент | Что обновить |
|---|---|
| Начал задачу | `Session.Goal`, `Session.Now` |
| Завершил шаг | добавь в `Session.Done`, обнови `Session.Now`, `Session.Next` |
| Ошибка решена | добавь в `Session.Errors` |
| Тронул файл | добавь в `Current.Modified` |
| Сменил ветку | обнови `Current.Branch` |

### dead-ends/<тема>.md — при провале подхода

Когда подход не сработал (после 2+ попыток или явный тупик):

```markdown
## <краткое название подхода>
Date: <дата>
Approach: <что пробовали>
Why failed: <почему не работает>
Lesson: <что делать вместо этого>
```

Файл называется по теме: `auth.md`, `database.md`, `websocket.md`.
Если файла нет — создай. Если есть — допиши.

**КРИТИЧНО:** НЕ жди конца сессии. Записывай СРАЗУ как понял что не работает.
Через час ты забудешь. Через сессию — точно забудешь.

### decisions.md — при значимом выборе

Когда выбираешь между подходами и это влияет на архитектуру:

```markdown
## <что решили>
Date: <дата>
Context: <почему встал вопрос>
Considered: <какие варианты рассматривали>
Decision: <что выбрали и почему>
Revisit if: <при каких условиях пересмотреть>
```

### status.md — при изменении состояния

- Тест прошёл → добавь в `## Working`
- Тест упал → добавь в `## Broken`
- Что-то заблокировано → добавь в `## Blocked`, обнови index.md `Blocked: да`

### journal.md — в конце сессии

```markdown
## <дата> — <что делали>
Duration: <примерно>
Did: <что сделано, 2-4 пункта>
Result: <итог>
Next: <что дальше>
Files: <ключевые файлы>
```

Когда записей >7 — перенеси старые в `journal-archive/YYYY-MM.md`.

## Формат index.md

```markdown
# Project: <название>
<цель проекта в одно предложение>

## Stage
Phase: <фаза> | Progress: <X/Y или %>
Blocked: <нет | да — см. status.md>

## Current
Task: <текущая задача>
Branch: <ветка>
Modified: <файлы тронутые в этой сессии>

## Session (live)
Started: <время>
Goal: <цель этой сессии>

Done:
- <что сделано>

Now: <что делаем прямо сейчас>
Next: <что будет следующим>

Errors:
- <ошибка — как решена>

## Last Session
<дата> — <1-2 строки>

## Docs
- status.md — что работает, что сломано
- decisions.md — технические решения
- dead-ends/ — провальные подходы по темам
- journal.md — последние сессии
```

## Перечитывание при сжатии контекста

Когда чувствуешь что потерял контекст (не помнишь что делал раньше
в этой сессии, не уверен в текущей задаче, не помнишь предыдущие шаги):

1. Перечитай `docs/index.md` — там `Session (live)` секция
2. В `Session.Done` — что уже сделано
3. В `Session.Now` — что делаешь
4. В `Session.Errors` — что уже ломалось и как починили
5. Продолжай с того места где `Session.Now`

**Это главная защита от потери контекста.** Обновляй index.md после
каждого значимого шага — если контекст сожмётся, ты восстановишься.

## Антипаттерны

НИКОГДА:
- Не пропускай чтение index.md — "я и так помню" = нет, не помнишь
- Не откладывай запись dead-ends — "запишу потом" = не запишешь
- Не читай ВСЕ файлы docs/ разом — читай по ссылкам
- Не начинай чинить баг не проверив dead-ends/ по теме
- Не предлагай подход не проверив decisions.md

ВСЕГДА:
- Читай index.md перед КАЖДЫМ ответом
- Обновляй Session после КАЖДОГО шага
- Пиши в dead-ends СРАЗУ при провале
- Перечитывай index.md если чувствуешь потерю контекста
