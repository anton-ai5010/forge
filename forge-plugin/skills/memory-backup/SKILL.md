# Memory Backup — память проекта в git

`.forge/` — вся память проекта: задачи, планы, идеи, решения, тупики, журнал. Пока она только на диске — она умирает вместе с диском. Этот скилл кладёт её в git и на GitHub.

## Процедура

1. Запусти:
   ```bash
   bash "$CLAUDE_PLUGIN_ROOT/skills/memory-backup/backup.sh" "<3-6 слов: что сохраняем>"
   ```

2. Разбери вывод — строки `FORGE-MEMORY:` адресованы тебе, не пользователю:

   - **«сохранена на GitHub»** → пользователю одна строка: *«Память проекта сохранена на GitHub.»* Внутри итога сессии или мержа — просто строка в отчёте, без отдельного сообщения.
   - **«push не прошёл»** → одна строка: сохранено локально, отправка не удалась (сеть или права). Работу НЕ блокируй, не зацикливайся на ретраях.
   - **«удалёнки нет»** → предложи: *«У проекта нет копии в облаке — если с компьютером что-то случится, вся история пропадёт. Создать приватный репозиторий на GitHub (видишь только ты)? Одна команда, ничего настраивать не надо.»* Только после явного «да»:
     ```bash
     gh repo create <имя-папки-проекта> --private --source=. --push
     ```
     Нет `gh` или пользователь отказался — не настаивай, память остаётся в локальных коммитах, повтори предложение не раньше следующей сессии.
   - **«нужна миграция»** → легаси-проект, `.forge` под игнором. Спроси: *«Включить сохранение памяти проекта в git? Задачи, планы и решения станут частью репозитория — в приватном это норм.»* После «да»:
     1. Убери строку `.forge/` (или `.forge`) из корневого `.gitignore`.
     2. Если репозиторий ПУБЛИЧНЫЙ — сначала просмотри `.forge/` на секреты (`grep -riE "password|пароль|token|secret|api.?key" .forge/`); найденное покажи пользователю до коммита.
     3. Запусти `backup.sh` снова — он сам создаст `.forge/.gitignore` со служебным мусором и закоммитит остальное.

## Правило секретов

В `.forge/` НИКОГДА не пишутся пароли, токены, ключи, приватные адреса — только названия записей Bitwarden («ключ лежит в Bitwarden: имя-записи»). Это правило действует при любой записи памяти, не только при бэкапе.

## Когда вызывается автоматически

- `session-awareness` — в конце итога сессии
- `finishing-a-development-branch` — после мержа
- по слову пользователя («сохрани», `/forge:save`)

## Что не попадает в git

`backup.sh` сам создаёт `.forge/.gitignore` со служебным мусором: `.inject-state`, `.last-backup`, `state.yml`, `.github-*`, `graph.json`. Всё остальное в `.forge/` — ценность, коммитится.
