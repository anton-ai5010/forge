# forge

Плагин для Claude Code. Превращает "я хочу X" в готовый код через **6-фазный пайплайн**, который сам ведёт тебя за руку.

Визуальная схема пайплайна: [ideas/pipeline-v2.html](../ideas/pipeline-v2.html)

## Как это работает

Ты пишешь любое "хочу / нужно / добавь / почини" — плагин подхватывает и проводит через фазы:

| Фаза | Что делает |
|------|-----------|
| 🎯 **new-task** | Уточняет задачу. Спрашивает только то на что можешь ответить только ты. Технику ищет сам в коде. |
| 🔬 **refine-idea** | Разбирает саму идею до плана: та ли проблема, нет ли пути проще, что сломает. На каждую слабость — как лучше. Потому что идея не всегда разумна. |
| 📋 **plan** | Строит план с чекпоинтами на смысловых границах. |
| 🔍 **critique** | 4 разных критика параллельно рвут план на части — Скептик / Прагматик / Архитектор / Адвокат пользователя. |
| 🚀 **execute** | Реализует. Тяжёлую работу делает в субагентах чтобы не загаживать чат. Останавливается только в чекпоинтах. |

После каждой фазы — твоё "ОК" или правка. Можешь сказать "стоп" в любой момент.

## Кому это нужно

Если пишешь код через Claude Code и тебя бесит:
- что Claude бросается кодить без понимания задачи
- что он суммаризирует в конце что и так видно в diff
- что задаёт технические вопросы которые ты не знаешь
- что плодит варианты A/B/C вместо одного решения
- что забывает контекст между сессиями
- что код "вроде работает но непонятно"

Плагин это лечит через структурный pipeline + 30+ скиллов под конкретные задачи (debugging, дизайн, тесты, рефакторинг и т.д.).

## Установка

```
/plugin marketplace add anton-ai5010/forge-plugin
/plugin install forge@forge-plugin
```

Перезапусти Claude Code. Плагин активен.

## Как начать работу

Просто напиши что хочешь. Например:
- "хочу добавить поиск по заметкам"
- "почему не работает кнопка"
- "сделай страницу красивее"

Плагин дальше сам.

**Полезные команды:**
- `/forge:unblocker` — если застрял
- `/forge:product-map` — увидеть всё что есть в проекте (HTML картинка)
- `/forge:explain X` — понять как работает X (HTML картинка)
- `/btw <вопрос>` — встроенная команда Claude Code: side-вопрос, ответ в overlay, не загаживает чат

## Опционально — Statusline

Чтобы внизу терминала показывалась текущая фаза + ветка + % контекста, добавь в `~/.claude/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "<путь-к-плагину>/forge-plugin/hooks/statusline.sh",
    "refreshInterval": 2
  }
}
```

## Что под капотом

- **L0/L1/L2 контекст** — плагин помнит структуру твоего проекта в `.forge/` и сам подгружает только нужное
- **Output Style "Forge Concise"** — заставляет Claude отвечать коротко и по делу (нативный механизм Claude Code, авто-активируется при включении плагина)
- **PreToolUse bash safety** — блокирует опасные команды (`rm -rf /`, `git push --force main`, etc.)
- **GitHub синхронизация** — Issue per task + Sub-issues per step + Pinned Issue с картой проекта + шапка в README. Включается флагом `github_sync: true` в `.forge/index.yml` или плагин сам предложит при первой задаче. Управление целями — `/forge:roadmap`.
- **Звуки** — mp3 на завершении ответа и запросе разрешения
