## Шаг 1: Стратегический слой `.forge/direction.yml` (для Клода)

**Файлы:**
- Изменить: `forge-plugin/commands/init.md` — добавить `direction.yml` в создаваемую `.forge` структуру + запись в `catalog:` L0

**Что делаем:** формат (для Клода, gitignored, машинный слой петли):
```yaml
# Цель и стадия НЕ дублируются — канон в index.yml (goal/stage, они в L0).
# Здесь только стратегия которой нет больше нигде.
directions:          # все направления к цели, кратко
  - name: "<короткое название>"
    why: "<одна строка: зачем / почему приоритетнее на этой стадии>"
    kind: hypothesis | blocker | build   # гипотеза к проверке / блокер / постройка
    priority: 1       # порядок по текущей стадии
backlog:             # отложенные направления
  - name: "<название>"
    why_deferred: "<кратко почему не сейчас>"
goal_shift:          # лёгкая история смещения цели (не полный аудит)
  - date: 2026-06-28
    from: "<была>"
    to: "<стала>"
    why: "<что узнали>"
```
Зарегистрировать в `index.yml` `catalog:` с тегами `[direction, goal, strategy, hypotheses, backlog, navigate, where-to-go]` — иначе forge-context/forge-start его не подхватят (находка Архитектора). Добавить и в шаблон `init.md`, и в живой `.forge/index.yml` этого проекта.

**Как проверим:** `direction.yml` валиден как YAML; есть запись в `catalog:`; цель/стадия в нём НЕ дублируются.

## Шаг 2: Переделать `project-unblocker/SKILL.md` в навигатор

**Файлы:**
- Изменить: `forge-plugin/skills/project-unblocker/SKILL.md`

**Что делаем (точечно):**

1. **Триггеры (description):** + «что дальше по проекту», «какие задачи», «направление», «куда двигаемся», «что делать с проектом». «Застрял» оставить.

2. **Профиль — оставить в скилле, обновить** свежим синтезом из фазы изучения (режимы провала: невход, точка пустого выбора, затык на 70-90%, побег в новый проект, прокрастинация-под-видом-подготовки, самообвинение; рычаги: физический микрошаг, готовый план, прямота без лести, один фокус). НЕ выносить глобально (потребитель один — отложено в задачу про детектор). Generic-ядро поведения («не знаю = реши за меня», «варианты вместо открытых вопросов») сохранить.

3. **Топливо** (автозагрузка): добавить `learnings.yml`, `library/*/spec.yml`, `direction.yml` к читаемым status/dead-ends/decisions/journal.

4. **Цель — живая, без трения.** Каждый запуск ПОКАЗЫВАТЬ цель строкой-якорем из `index.yml.goal` («Цель: X. Стадия: Y.»), НЕ вопросом. Спрашивать «сместилась?» ТОЛЬКО когда: цели нет / сменилась стадия / находки противоречат цели. При смещении — обновить `index.yml.goal` (канон в L0) + дописать `goal_shift` в `direction.yml` + строку в `journal.yml`. Направление синхронизации одностороннее: index.yml ← навигатор.

5. **Мышление гипотезами.** Направления формулировать как гипотезы/проверки/блокеры между «сейчас» и «целью», писать в `direction.yml.directions`.

6. **Фаза 4 «План» → расписать ВСЁ кратко + рекомендация-дефолт.** Главная правка под коррекцию Антона:
   - Расписать ВСЕ направления (не топ-3 — Антон хочет видеть все), каждое КРАТКО: название + одна строка «зачем/почему приоритетнее». НЕ простынёй.
   - Приоритет — с учётом стадии (`index.yml.stage`): ранняя → проверка гипотез; поздняя → доводка/деньги.
   - Рекомендация = ДЕФОЛТ-движение, не вопрос на подумать: «Иду по [направление 1] — вот почему. Не туда — скажи, разверну другое.» Молчание / «не знаю» / «давай» = идём по рекомендации.
   - **Согласовать таблицу «Обработка жалоб» (строки ~184-193)** с этим: «не знаю» = «фиксирую свою рекомендацию как выбор» (а не зависнуть в ожидании). Снять противоречие «выбери за него ОДНУ» vs «ждать корректировку».
   - Антипаттерн против простыней СОХРАНИТЬ, но переформулировать: не «макс 3 задачи», а «каждое направление кратко, не абзац; видны все».

7. **Фаза 5 «Запись» → петля + хэндофф.** Писать `direction.yml` (для Клода: directions/backlog/goal_shift) + `ROADMAP.md` (для Антона: все направления человекочитаемо, кратко). Заканчивать ОДНИМ физическим шагом, поданным как действие (глагол+объект), и **авто-handoff**: «Запускаю `/forge:new-task` на это» (в стиле остального пайплайна), а не «дальше сам».

**Как проверим:** поведенческий прогон — цель строкой-якорем; все направления кратко с приоритетом-по-стадии; рекомендация-дефолт; пишет direction.yml + ROADMAP; заканчивается физическим шагом + предложением new-task.
