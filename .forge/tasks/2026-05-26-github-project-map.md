# GitHub Project Map — синхронизация пайплайна и карты проекта с GitHub

## Задача

Внедрить в forge-plugin **два уровня синхронизации с GitHub**, оба односторонние (плагин → GitHub, Claude может читать обратно для контекста):

1. **Задачи** — каждая задача из 4-фазного пайплайна автоматически создаётся как Issue + Sub-issues и закрывается по мере прохождения фаз.
2. **Карта проекта** — три поверхности на GitHub постоянно держат пользователя в курсе "где он сейчас в проекте"; обновляются плагином автоматически после каждой завершённой сессии.

Пользователь ничего руками в GitHub не делает — только смотрит готовую картинку. Главная цель — лечить боль "забываю и теряю контекст проекта" между сессиями.

## Критерий готовности

### Поверхности на GitHub (когда пользователь заходит в репо проекта)

- В шапке README.md сразу видна секция с 3 строками: `🧭 Сейчас / ⏭️ Следующее / ✅ Недавно сделал` — отражает актуальное состояние из `.forge/index.yml` и последней записи в `.forge/journal.yml`
- В списке Issues наверху прибит Pinned Issue "🗺 Карта проекта" — содержит **полный список** крупных целей проекта (без отсечки по количеству), отсортированный по приоритету; у каждой цели — название, краткое описание, прогресс-бар вида "3 из 12 задач готово"
- В репо существует Projects v2 board с колонками `Сейчас / Следующее / Потом / Готово`; карточки = milestones (крупные цели), ездят между колонками автоматически по мере работы

### Поведение пайплайна

- После `/forge:new-task` — создаётся GitHub Issue: заголовок = название задачи, тело = задача + критерий чекбоксами, label `forge:phase-1`; Claude **сам подбирает подходящую цель** из карты по смыслу задачи и привязывает Issue к соответствующему milestone; если ни одна цель не подходит — создаёт новую цель; одной строкой пишет в чат что сделал ("привязал к цели X" или "создал цель Y")
- Если пользователь говорит "переcyдь на цель Z" — Claude переcyживает Issue к другому milestone
- После `/forge:plan` — под Issue создаются Sub-issues на каждый шаг плана; label родителя → `forge:phase-2`
- После `/forge:critique` — label → `forge:phase-3`, добавляется комментарий с найденными блокерами и важными правками
- Во время `/forge:execute` — Sub-issues закрываются по мере выполнения; когда все закрыты — родительский Issue закрывается; label → `forge:done`
- После закрытия Issue Claude автоматически обновляет: (1) шапку README.md, (2) прогресс-бар цели в Pinned Issue, (3) статус карточки в Projects board, (4) `.forge/journal.yml`

### Команды

- `/forge:roadmap` — для ручного управления картой целей: добавить новую цель, переименовать, переупорядочить приоритет, удалить, перенести между колонками
- `/forge:start` — в новой сессии читает не только локальный `.forge/`, но и Pinned Issue из GitHub; показывает дашборд "вот где мы остановились"

### Граничные условия

- Если в проекте нет GitHub remote или флага `github_sync: true` в `.forge/index.yml` — фича молча выключена, плагин работает как раньше
- Приватная локальная память **не дублируется** в GitHub: `dead-ends.yml`, `learnings.yml`, `decisions.yml`, `journal.yml` остаются только в `.forge/`
